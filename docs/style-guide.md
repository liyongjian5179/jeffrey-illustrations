# Style Guide

## 风格定义

Jeffrey Warm Tech Illustration：

- 温暖手绘
- 局部细腻蜡笔 / 彩铅质感
- 粗线条
- 高留白
- 蓝黄品牌色
- 轻松但不幼稚
- 技术感来自动作、物件和隐喻，不靠黑客氛围

## 色彩锁定

本地生图如果没有把参考图作为输入图传入，必须在 prompt 里显式写入色值：

- Hoodie 主色：`#5379E8`
- 耳机 / 重点描边：`#2F56C5`
- 黄色 Logo / 点缀：`#FFD84A`
- 背景奶油白：`#FFF9EE`
- 牛仔裤：`#88A8F5`
- 头发 / 线稿：`#2B2F36`

避免紫蓝 hoodie、青蓝耳机、橙黄 Logo、灰底、棕发和随机高饱和点缀色。

## 质感范围

- Jeffrey、Byte、衣服、头发、道具和粗线条可以有轻微蜡笔 / 彩铅颗粒。
- 背景保持干净、平整、奶油白，不要整张图都糊一层颗粒。
- 中文标签、卡片、UI-like 面板要清晰可读，只保留轻微手绘线条质感。
- 阴影只做少量淡铅笔排线。

避免重颗粒全图覆盖、脏纸纹、文字糊掉、卡片区域噪点过重。

## 构图原则

- 一张图只表达一个核心观点
- 主体占画面 40%–60%
- 背景尽量干净
- 中文标注尽量短
- 不要复杂 UI、复杂架构图或密集文字
- “规范 / 原则 / 系统 / 标准”这类抽象主题，也要先转成一个动作隐喻，不要默认画满白板卡片

## 构图模式

### Warm Article Mode

适合公众号、博客和叙事型正文配图。

- 可以保留少量创作者语境道具，例如笔记本、咖啡杯、绿植、笔筒、桌面工具。
- 道具必须服务温暖创作者场景。
- 道具不能抢走核心隐喻，也不能让画面变成杂物桌。

### Clean Skill Mode

适合 README、GitHub 文档、Skill 示例、Notion / Obsidian 笔记和方法论图。

- 只保留隐喻所需物件。
- 避免无关桌面装饰。
- 留白更多，背景元素更少。

## 必须保留

- 黑色或深蓝黑 Jeffrey Hair™
- Jeffrey Blue Hoodie
- 黄色 J Logo
- 圆框眼镜
- 蓝色耳机
- Byte（当场景需要互动时）

## 禁止

- 赛博朋克
- 黑客暗色终端墙
- 商务精英风
- 西装
- 胡子
- 棕发
- 半写实成人头像
- 角色设定板式构图，除非明确要求
- 白板上堆很多文字卡片或 UI 模块
- 大号悬浮球形 Byte
- 儿童绘本过度幼稚
- 3D 渲染
- 复杂阴影 / 渐变 / 写实材质
