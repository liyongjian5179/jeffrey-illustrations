# Composition Patterns

Use these patterns to convert abstract technical ideas into visual scenes.

## 1. Workflow path

Jeffrey and Byte move through a simple path, conveyor belt, pipeline or stepping stones.

Best for: AI Agent, DevOps pipeline, publishing workflow.

## 2. System local view

Show one meaningful part of a system instead of the whole architecture.

Best for: container, queue, cache, server, MCP.

## 3. Before / after

Two clean halves showing change.

Best for: refactor, automation, migration, before AI / after AI.

## 4. Character state

Jeffrey's posture expresses a technical state.

Best for: debugging, stuck, breakthrough, deployment success.

## 5. Physical metaphor

Turn an abstract idea into a tangible object.

Best for: knowledge graph, trust, context, prompt, memory.

## 6. Mini comic

2–3 panels, no more.

Best for: small story, problem -> attempt -> success.

## Avoid spec-board drift

For abstract topics such as "visual guidelines", "system standards", "rules" or "principles", do not default to a whiteboard full of cards and text.

Instead, turn the topic into a physical action:

- Jeffrey aligns two puzzle pieces.
- Byte hands over one checklist card.
- Jeffrey stamps one approved note.
- Jeffrey compares two simple blocks.
- Byte highlights one key object.

Keep labels to 2-3 short phrases. If the image starts looking like a PPT / UI board, simplify it into a single metaphor.
